# DEFINITIVE SPECIFICATION: VIRALSCOPE AI

## 1. Context and Objectives
*   **Main Objective**: Develop a multimodal Deep Learning model capable of predicting the **Reach Multiplier** of a YouTube video (its potential to outperform the channel's usual audience) by analyzing its thumbnail and title.
*   **Architectural Approach**: Natively multimodal neural network using **CLIP** (Contrastive Language-Image Pre-training) to align visual and semantic features in a shared latent space.

## 2. Technical Stack (Production-Optimized)
*   **Language & Base Framework**: Python 3.10+, PyTorch 2.x.
*   **Multimodal Backbone**: Hugging Face `transformers` (**CLIP** — `openai/clip-vit-base-patch32`).
*   **Data Engineering**: Pandas, NumPy, PIL, CLIPTokenizer.
*   **Interpretability (XAI)**: **Captum** (Integrated Gradients) and Patch-based **Grad-CAM** for Vision Transformers (ViT).
*   **Deployment (MVP)**: **Gradio** interface with native PyTorch model.

## 3. Development Steps (Technical Pipeline)

### Phase 1: Data Engineering & Bias Resolution (Data Prep)
*   **1.1. Collection**: Use **US + GB + CA datasets only** (English-speaking markets: `USvideos.csv`, `GBvideos.csv`, `CAvideos.csv` from Kaggle YouTube Trending Dataset).
*   **1.2. Target Engineering (The "Reach Multiplier")**:
    *   To counter *Fame Bias* and avoid *Data Leakage*, channel size will **not** be an input variable.
    *   The *Target* will be calculated as: `Multiplier = Video_Views / Channel_Average_Views`.
    *   **Filtering Unreliable Channels**: Channels with `< 3 videos` are excluded.
    *   Binarization (Classification): Target `1` if Multiplier > 1.5, otherwise `0`.
*   **1.3. I/O Optimization**: Upstream thumbnail filtering is MANDATORY. Records without valid thumbnails are dropped before the train/test split. Text is pre-tokenized using the **CLIPTokenizer** (max_length=77) to avoid CPU bottlenecks.
*   **1.4. Grouped Splitting & Weights**: Train/Val/Test split based on **`GroupShuffleSplit`** grouped by `channel_id`. Class imbalance is handled via **`WeightedRandomSampler`** and `BCEWithLogitsLoss`, focusing on ranking rather than naive accuracy.

### Phase 2: Vision Modeling (CLIP Branch)
*   **2.1. Feature Extraction**: Integrate CLIP's **Vision Transformer (ViT-B/32)**. Generates L2-normalized 512-dim visual embeddings.
*   **2.2. Image Pipeline**: Resize to `224x224` with CLIP-specific normalization (`mean=[0.4814, 0.4578, 0.4082], std=[0.2686, 0.2613, 0.2757]`).

### Phase 3: NLP Modeling (CLIP Branch)
*   **3.1. Semantic Architecture**: Integrate CLIP's **Text Transformer**. Generates L2-normalized 512-dim semantic embeddings in the same latent space as the image.
*   **3.2. Cross-Modal Alignment**: Leverages CLIP's pre-trained ability to compute semantic similarity between images and text.

### Phase 4: Interaction Fusion & Training
*   **4.1. Fusion Logic**: Instead of naive concatenation, the model computes cross-modal interaction features:
    - **Element-wise Difference**: Captures modal contradiction.
    - **Element-wise Product**: Captures semantic reinforcement.
    - **Cosine Similarity**: Direct alignment signal in the shared space.
    - Final Input: `[img_emb, txt_emb, diff, prod, cos_sim]` (Total: 2049 dimensions).
*   **4.2. Strategy**: Two-stage fine-tuning. Head warmup (Stage 1) followed by full backbone-unfreezing (Stage 2) with staggered learning rates ($10^{-3} \to 10^{-4}$ for the head, $5 \times 10^{-6}$ for the CLIP backbones).
*   **4.3. Loss**: `BCEWithLogitsLoss` (optimized for PR-AUC to handle imbalanced trending labels).

### Phase 5: Interpretability (XAI) & MLOps
*   **5.1. ViT-Based Grad-CAM**: Applied specifically to the final encoder layer of the CLIP Vision Transformer. Reshapes patch tokens back to a spatial grid to generate heatmaps on thumbnails.
*   **5.2. Integrated Gradients**: Applied to CLIP Text embeddings to attribute viral potential to specific keywords.
*   **5.3. Deployment**: Create a **Gradio** user interface encapsulating the compiled multimodal weights. No ONNX export used to maintain XAI compatibility.

## 4. Final User Experience (Creator SaaS)
The tool becomes an instant "Diagnostic Dashboard":
1.  **Input**: The creator uploads their thumbnail and types their title.
2.  **Output (Business Score)**: The model returns a viral score relative to the training distribution.
3.  **Output (XAI Diagnostic)**:
    *   The thumbnail is displayed with a Heatmap locating the area of high visual impact.
    *   The title is displayed with color coding generated by *Integrated Gradients*.
